Getting Started

New to the terminal? No problem.

Follow these five steps and you'll have a fully configured AI development environment in under an hour.

1

Open Terminal

Press Cmd + Space, type Terminal, and press Enter. Terminal is the app that lets you type commands directly to your Mac.

2

Copy and paste the install command

Click the Copy button above, then paste it into Terminal with Cmd + V and press Enter.

3

Enter your Git details

The script will ask for your name and email address. Use the same ones as your GitHub account. These appear on your code commits.

4

Choose your tools

A checkbox menu appears for each category — AI tools, databases, editors, and apps. Use ↑/↓ to move, Space or Enter to toggle a tool on/off, and D to confirm your choices. Not sure? Just press D to accept the recommended defaults.

5

Wait while everything installs

The script downloads and installs only the tools you selected. This can take 20–60 minutes depending on what you chose and your internet speed. You can walk away — just stay nearby in case your Mac asks for your password.

6

Restart your terminal and follow the next steps

When the script finishes, close the terminal window, open a fresh one, and follow the After Install checklist below.

# Skip all menus — install every default, no interaction bash <(curl -fsSL https://raw.githubusercontent.com/otto-ai-labs/setupai.dev/main/setup.sh) --yes
# Just the essentials — languages and shell tools only bash <(curl -fsSL https://raw.githubusercontent.com/otto-ai-labs/setupai.dev/main/setup.sh) --minimal
# Skip the AI tools category entirely bash <(curl -fsSL https://raw.githubusercontent.com/otto-ai-labs/setupai.dev/main/setup.sh) --skip-ai-tools
# Skip local databases bash <(curl -fsSL https://raw.githubusercontent.com/otto-ai-labs/setupai.dev/main/setup.sh) --skip-databases
# Skip web/JS tools entirely bash <(curl -fsSL https://raw.githubusercontent.com/otto-ai-labs/setupai.dev/main/setup.sh) --skip-web
Flag What it does
--yes, -y Skips all menus — installs every default-selected tool with no interaction. Ideal for re-runs.
--minimal Skips all menus entirely. Installs only languages and shell tools.
--skip-ai-tools Skips the AI tools category entirely (no menu shown).
--skip-databases Skips the databases category entirely (no menu shown).
--skip-web Skips all web/JS tools — pnpm, TypeScript, ESLint, Prettier, Vite, Bruno, and more.
--help Shows all available options in the terminal.
After Install

You're almost ready to build.

Complete these steps after the script finishes and you'll be set up to build real AI applications.

Reload your terminal

Close the terminal window and open a fresh one, or run source ~/.zshrc to load the new configuration.

Add your SSH key to GitHub

The script generated a key for you. Copy it with cat ~/.ssh/id_ed25519.pub and add it at github.com/settings/keys.

Set up your Anthropic API key

Get a key at console.anthropic.com, then add it to your shell: export ANTHROPIC_API_KEY='sk-...'

Set up your OpenAI API key (optional)

Get a key at platform.openai.com, then: export OPENAI_API_KEY='sk-...'

Run a local AI model with Ollama

Try running Llama 3 entirely on your Mac — no API key needed: ollama run llama3

Launch Jupyter and start experimenting

Open an interactive notebook environment: jupyter lab (or use the shortcut alias jl)

Restart your Mac

A restart ensures all system-level changes (animations, performance tweaks) take full effect.

Security & Transparency

What the script does to your system

Full transparency — no hidden changes. Review the scripts yourself before running.

Three scripts are downloaded and run from the internet

Setup installs Homebrew, nvm, and Oh My Zsh using their official install scripts (curl | bash). Review them at Homebrew, nvm, and Oh My Zsh before running.

SSH key generated without a passphrase

For a smooth non-interactive install, the SSH key is created with -N "" (no passphrase). To add one, open scripts/modules/git.sh and remove -N "" before running.

macOS app quarantine is disabled

osx.sh sets LSQuarantine = false, which removes the "Are you sure you want to open this application?" dialog for downloaded apps. Remove this line from osx.sh if you prefer to keep the protection enabled.

All source code is public and auditable

Every script is readable on GitHub. Start with setup.sh to see exactly what runs and in what order.
